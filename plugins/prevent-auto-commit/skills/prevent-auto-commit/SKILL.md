# prevent-auto-commit

git commit または jj new を実行する前に、この実行が承認されているかを確認する。

## 実行が許可されるケース

以下のいずれかに当てはまる場合のみ、git commit または jj new を実行してよい：

1. **ユーザーの明示的な指示**: 現在の会話でユーザーが直接「git commitして」「jj newして」「コミットして」「変更を保存して」「スナップショットを作って」などと指示した
2. **スキル・コマンドのワークフロー**: 現在実行中のスキルやコマンド（例: `jj-new` スキル、`commit-push` スキル、`commit-commands:commit-push-pr` スキルなど）の手順の一部として、git commit や jj new の実行が明示的に組み込まれている

## 実行してはいけないケース

以下の場合は git commit および jj new を実行してはならない：

- 作業（コード修正、ファイル編集、タスク完了）が終わったからといって、AIが自己判断でコミットやスナップショットを作成する
- 「変更を整理するために」「記録として残すために」という自己判断
- ユーザーに指示されていないのに、作業の締めくくりとしてコミット・スナップショットしようとする

## 判断フロー

git commit または jj new を実行しようとしたとき：

1. この会話の直近のユーザーメッセージに、明示的な実行指示が含まれているか？
2. 現在実行中のスキル・コマンドの手順として組み込まれているか？

**どちらも No の場合 → 実行を中止する**

ユーザーに以下のように伝える：

> 作業が完了しました。変更はまだコミット（またはスナップショット）していません。
> コミットを作成する場合は「git commitして」「jj newして」などとお知らせください。

## なぜこのルールが必要か

コミットとスナップショットはユーザーが意図的に行うアクションであるべきで、AIが「作業完了の印」として自動実行するものではない。AIが勝手にコミットすると、ユーザーが意図しない粒度・タイミング・メッセージでgit履歴が作られ、後から整理が難しくなる。
